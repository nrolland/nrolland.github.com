An introduction through dictionary

You have a list of N things, to which you'd like to associate some elements.

A linear dictionary

You could build a dictionary based on a linear search declaring the type as follow, where we wrap our underlying type in a constructor to make it opaque.

1: type LDict<'K,'V when 'K:equality> = LDict of List<'K*'V>

To act on this type, we declare some functions which we group under a Module nmed after the type. Now an external user can cuse our librabry without our implementation details leaking out to him.

 1: module LinearDictionary =
 2:     let empty :LDict<'T,'V> = LDict List.empty
 3:     let add (LDict dic:LDict<'T,'V>) (key,value)  : LDict<'T,'V>  =  LDict ((key,value)::dic)
 4:     let create keys values = Seq.fold add empty (Seq.zip keys values)  
 5:     let find (LDict dic:LDict<'T,'V>) (key:'T) = 
 6:         List.tryPick(fun (k,v) ->  if k = key then Some v else None) dic
 7:     let count (LDict dic:LDict<'T,'V>) = dic.Length 
 8: 
 9: let l1 = [1;2;3;4]
10: let l2 = [10;20;30;40]
11: 
12: let d = LinearDictionary.create l1 l2
13: let f = LinearDictionary.find d 2
14:

Obviously such approach has severe performance cost as every lookup is in o(N) We introduce a few helper methods to time things

 1: let duration f = 
 2:     let timer = new System.Diagnostics.Stopwatch()
 3:     timer.Start()
 4:     let returnValue = f()
 5:     float timer.ElapsedTicks
 6:     
 7: let rseq = let rng = System.Random() in Seq.initInfinite(fun _ -> rng.Next())
 8: let ints = Seq.initInfinite id
 9: let takeNSample N ls =  
10:     let l = List.length ls 
11:     ls |> List.mapi(fun i e -> if i%(l/N) = (l/N-1) then Some e else None) |> List.choose id
12: 
13: module List =
14:     let rec skip n ls = match n with | 0 -> ls | n -> skip (n-1) (List.tail ls)
15:     let split n ls = 
16:         let (hs,ts), n = List.fold(fun ((hs,ts),k) e -> if k < n 
17:                                                         then ((e::hs,List.tail ts),k+1) 
18:                                                         else ((hs,ts),k+1))  ((List.empty,ls), 0) ls                      
19:         hs |> List.rev, ts

Now if we make make 30 dictionaries of size ranging from 1000 to 2000

1: let NStart = 1000
2: let N, NSample = 1000, 30 
3: let lStart, lAdditional = List.split NStart (ints|>Seq.take(N+NStart)|>Seq.toList)
4:

6: let dics       = listOfKeys |> List.map (fun ks -> LinearDictionary.create ks rseq)

We can time how much it takes for a key selected at random in those Linear dictionaries

 1: let rng = System.Random()
 2: let std es = let l, sum = Seq.fold (fun (n,s) e -> (n+1, s + e)) (0, 0.) es
 3:              let avg = sum/(float l)
 4:              avg, Seq.sum (Seq.map (fun x ->  pown (x - avg) 2) es) / (float l)
 5: 
 6: 
 7: let NAverage =  50 //variability of average
 8: let NRepeat  = 100  //variability of estimation of variability of average
 9: let times  = dics |> List.map (fun d -> let l = LinearDictionary.count d
10:                                         l, List.init NRepeat (fun _ -> //we repeat
11:                                             (List.init NAverage (fun _ -> 
12:                                                 duration(fun()->LinearDictionary.find d (rng.Next l))) |> List.average) ))
13:                  |> List.map (fun (l, ts) -> std ts |> fst |> int )

We can see that we it takes a time linear in the size of the dictionnary to find the elements :

val NAverage : int = 50 val NRepeat : int = 100 val times : int list = [385; 446; 450; 461; 558; 554; 484; 459; 475; 519; 485; 519; 601; 535; 573; 650; 796; 850; 759; 595; 649; 682; 688; 740; 674; 717; 722; 771; 743; 768]

The time it takes to find a random key in a LinearDictionary is roughly twice as much for 2000 elements than it is for a 1000.

Actually we have two effects in our mesurements. The first is the proper time that it takes on average, which we estimate via a sample. The second is the noise due to the specific chosen sample : we could be lucky or unlucky in the random series of indexes to be searched. As variance is a quadratic function, additive in indepedant variables, the variance of the average measurement should decrease proportionnaly with the number of we average upon. That is $$\mathrm{Var}[ \frac{1}{N} \sum x_i ] = \frac{1}{N^2} \sum \mathrm{Var} [ xi ]= \frac{1}{N} \mathrm{Var} [x ] $$

Hashing

The idea of hashing is to pre-position all the items that were in one long list, into a set of $M$ bins. To do so, we use a function, from the set $S$ of size $N$, of possible input, to the range $1..M$

We only append element in those bin whose index is the hashed value of the element. This reduces the lists size, allowing to gain speed.

Mathematically, $x, y \in S -> h x = h y$ is an equivalency reationship, which alllows us to split the input space into $card Im h$ equivalency classes.

 1: type HDict<'K,'V when 'K:equality> = HDict of List<'K*'V> array
 2: 
 3: module HashDictionary =
 4:     let empty (size:int):HDict<'T,'V> = HDict (Array.create size List.empty)
 5:     let add (HDict dic:HDict<'T,'V>) (key,value) : HDict<'T,'V>  =  
 6:         let k = (hash key)%dic.Length
 7:         dic.[k] <- ((key,value)::dic.[k])
 8:         HDict dic
 9:     let create size keys values = Seq.fold add (empty size) (Seq.zip keys values)  
10:     let find (HDict dic:HDict<'T,'V>) (key:'T) = 
11:         List.tryPick(fun (k,v) ->  if k = key then Some v else None) (dic.[hash key%dic.Length])
12:     let count (HDict dic:HDict<'T,'V>) = Array.fold (fun s (e:List<_>) ->  (+) s e.Length) 0 dic

If we want to time our new code, we have to repeat much of what was done before, so we can usefully abstract over the moving parts

1: let time create count find listOfKeys = 
2:     let rng = System.Random()
3:     listOfKeys |> List.map (fun ks -> create ks rseq)
4:                |> List.map (fun d -> let l = count d
5:                                      l, List.init NRepeat (fun _ -> //we repeat
6:                                      (List.init NAverage (fun _ -> 
7:                                         duration(fun()-> find d (rng.Next l))) |> List.average) ))
8:                |> List.map (fun (l, ts) -> std ts |> fst |> int )

We can verify that it works on our original dictionary

1: let times2 = time LinearDictionary.create LinearDictionary.count LinearDictionary.find listOfKeys
val times2 : int list = [359; 381; 376; 385; 389; 403; 418; 446; 438; 472; 490; 470; 469; 493; 511; 529; 509; 556; 573; 569; 560; 614; 648; 595; 607; 659; 656; 651; 682; 662]

And get the numbers for our hash-based dictionary

1: let times3 = time (HashDictionary.create 100) HashDictionary.count HashDictionary.find listOfKeys
val times3 : int list = [4; 13; 4; 7; 5; 4; 8; 4; 8; 5; 5; 9; 5; 9; 5; 6; 5; 6; 5; 9; 6; 10; 6; 10; 6; 10; 6; 7; 6; 10]

You see how dreadfully effective this approach is at speeding things. Can we do better.

Randomization of hash functions, universal hashing

As hashing amounts to partitioning $S$, it is of no use if all the elements picked belong to the same equivalency class : all elements would point to the same bin. For any hash function this has to be the case if $card S > m$, but we can make this set non-determistic if we had some randomness to our hash function.

This property can be formalized in universal families. We want the property that if we choose $h$ at random in $H$, then $P(h x = h y) = \frac{1}{M}$ That means $|h \in H s.t. h x = h y| = card H * P(h x = h y) = frac{|H|}{m}$
